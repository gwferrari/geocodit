GeocodIT Language Profile

A language profile for GeocodIT semantic web application.

View GeocodIT project in GitHub

Abstract

GeocodIT is a system that manages italian geocoding using W3C semantic web best practices and standards.

In order to let computers to work for us, they must understand data. Not only grammar and syntax (like in EDI) but the real meaning of things. When this is clear, machines can recognize, manage data and make decisions even when their provenance and quality is not totally known and data are incomplete. To enable this, we need to share knowledge using a language that is understandable both for humans and machines.

A language profile is a trimmed down version or an extension of a set of ontologies that trades some expressive power for the efficiency of reasoning.

This document describes vocabularies and constraints to be used to describe GeocodIT data.

Name spaces

The following prefixes are defined inside this document

  • ter: http://datiopen.istat.it/odi/ontologia/territorio/
  • gco: http://linkeddata.center/ontology/geocodit/v1#
  • geo: http://www.w3.org/2003/01/geo/wgs84_pos#
  • rdfs: http://www.w3.org/2000/01/rdf-schema#

This document can be imported as a RDFa 1.1 profile.

GeocodIT ontology

The GeocodIT ontology defines some terms used to annotate linked data.You should understand all ontology terms.

Ontologia del territorio

The ontologia del territorio is published by ISTAT.

You should understand at least following terms:

  • ter:COM
  • ter:REG
  • ter:PROV
  • ter:regione_di_PROV
  • ter:provincia_di_COM
  • ter:haNome
  • ter:haCodIstat

All the properties must be considered restricted to cardinality =1 (ie. exactly a value for each attribute)

Basic Geo (WGS84 lat/long) Vocabulary

Geo is a basic W3C RDF vocabulary that provides the Semantic Web community with a namespace for representing lat(itude), long(itude) and other information about spatially-located things, using WGS84 as a reference datum.

RDF Schema 1.1

RDF Schema 1.1   provides a data-modelling vocabulary for RDF data. RDF Schema is an extension of the basic RDF vocabulary.

You should understand only following term:

  • rdfs:label used to indicate a common name for the object, only @it language tag should be used
  • rdf:comment used to annotate uri, no special meaning asociated

OWL

OWL provides an ontology description language, besides to describe vocaularies, you should use owl:sameAs to interlink data.
